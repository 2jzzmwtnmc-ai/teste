S/4HANA Cutover Command Center

Professional Execution Status Monitoring

1) What this dashboard is

The S/4HANA Cutover Command Center is a Qlik dashboard that provides real-time execution status of cutover activities. All data is managed in a central Excel file hosted on SharePoint.

Real-Time Monitoring

Identify delays and track progress live.

Refresh Rate

Dashboard updates every 5 minutes based on the latest saved Excel data.

2) Data source (Excel on SharePoint)

The system relies on two key tabs within the SharePoint Excel file:

  • Task List: Main database of cutover activities.
  • Phases / Schedule: Timeline definitions (Planned windows).

Reference: Phases / Schedule tab logic

3) Required Fields

Every task MUST have the following fields filled to ensure KPI accuracy:

Phase & Sub-Phase
Start Date & Time
End Date & Time
Progress (0-100%)
Country
UAT Flag (if app.)

Reference: Task List column layout

CRITICAL: Avoid these data errors

  • Start date later than end date.
  • Missing dates or missing hours.
  • Tasks placed in "Off dashboard" phases.

8) Executive Overview

Designed for a Left → Right and Top → Bottom reading flow.

Main Interface: Executive Overview Page

A) Filters (Context)

Located top-right. Choose Stage (UAT/Go-Live) and Country. This context resets all KPIs instantly.

B) Main KPIs

  • Total Progress: Weighted average (longer tasks carry more impact).
  • Time Elapsed: % of the window already consumed.
  • Countdown: Remaining time until the context end date.
  • Status: Automatically flagged as On Track, At Risk, or Late.

C) Action Table

The bottom section shows the "Overdue activities (Action Required)" list. Use this for daily follow-ups with owners.

9) Data Health & QA

Validation Tab

"If the data is wrong, the KPIs are wrong."

Reference: Data Quality Issues Report

Expected Action:

  1. Identify the task in the Data Quality Issues table.
  2. Open Excel source file.
  3. Correct the specific row (check dates, phases, or missing info).
  4. Save Excel and wait 5 minutes for refresh.

10) Best Practices

  • Update % Progress multiple times a day during critical windows.
  • Keep Start/End hours realistic to actual plan.
  • Use the "UAT = X" rule consistently.
  • Never leave required fields blank.

11) Quick Troubleshooting

"My task is missing from the dashboard"

Check: Is the Phase/Sub-Phase filled? Is it marked "Off dashboard"? Is the Country correct? Did you mark UAT with an "X"?

"Overdue count looks too high"

Check for typos in the end dates or tasks that are done but still show progress < 100%.
